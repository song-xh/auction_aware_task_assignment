# CAPA Phase 4 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Implement a paper-faithful CAPA Phase 4 code path covering CAMA, DAPA / DLAM, an end-to-end CAPA batch runner, TR / CR / BPT metrics, and unit/integration tests without implementing RL-CAPA.

**Architecture:** Add a new standalone `capa/` package instead of extending the legacy Chengdu scripts. Keep the new implementation generic and deterministic by injecting a simple travel-time / distance interface, so CAPA logic is reusable by later RL-CAPA work and testable without the Chengdu map loader.

**Tech Stack:** Python standard library, `dataclasses`, `typing`, `unittest`, repository-local Markdown docs.
